# De-AI the UI/UX of My Next.js App

You are a senior product designer and frontend engineer with taste. I'm giving you a Next.js application that *looks like it was generated by an AI* — and I want you to make it feel like it was crafted by a human designer with opinions.

## What "AI-ish" means (eliminate these on sight)

**Visual clichés:**
- Purple-to-blue / indigo-to-pink gradients, especially on hero sections
- Glassmorphism / frosted-glass cards layered over gradient blobs
- `rounded-2xl` + `shadow-lg` + `bg-white/80` on every card
- Sparkle ✨ icons, gradient text on headlines, "Powered by AI" energy
- Lucide icons used as decoration next to every heading
- shadcn/ui components used with zero customization
- Tailwind's default `slate` / `zinc` / `gray` palette, untouched
- Inter (or Geist) at 4 weights, applied uniformly everywhere
- Bento grids where every cell is the same card component
- Three-column "Features" section with icon + heading + one sentence
- Centered hero: tagline → subtagline → two CTA buttons → screenshot mockup

**Layout clichés:**
- Everything centered, perfectly symmetric, evenly spaced
- Identical vertical rhythm between every section (96px / `py-24`)
- Container max-width 1280px, content always centered within it
- Section → section → section with no editorial flow
- Excessive whitespace masquerading as "minimalism"

**Copy clichés:**
- "Build faster. Ship smarter." style taglines
- "Get started" / "Learn more" as the only two CTAs
- Vague benefit-driven headers with no specificity
- Em-dashes used decoratively in marketing copy
- Lorem ipsum or generic placeholder copy still present

**Motion clichés:**
- `fade-in-up` triggered on every section as you scroll
- Framer Motion used identically for every animated element
- Hover states that just scale to 1.05

## Your task — work in this order

### 1. Audit
Walk the codebase and produce a **specific list** of AI-ish patterns you find. Quote the actual class strings, component names, and file paths. Don't generalize — point at lines.

### 2. Establish a real design language
Before touching components, decide:
- **A point of view.** Is this app editorial? Brutalist? Swiss/grid? Tactile/textured? Late-90s utilitarian? Pick one and commit. State it in one sentence.
- **A palette of 5–7 colors max**, with at least one unexpected accent (mustard, oxblood, terracotta, electric lime — not "indigo-500"). Define them as CSS custom properties or extend Tailwind's theme. Remove the default palette from use.
- **A type system with character.** Pair fonts intentionally — e.g., a serif display face (Fraunces, Instrument Serif, GT Sectra) with a workhorse sans, or a mono for accents. Use real type scale ratios, not Tailwind defaults. Vary weight and size dramatically across the page.
- **A spacing rhythm that isn't uniform.** Some sections breathe, others are dense. Asymmetric is fine. Tight is fine.

### 3. Rewrite components with intent
- Replace generic cards with layouts that fit the *content* — sometimes a list, sometimes a quote pull, sometimes a table, sometimes just a paragraph with a number next to it.
- Kill the bento grid unless it's genuinely the right shape for the data.
- Replace icon+heading+sentence feature blocks with prose, anecdotes, or screenshots that show the thing working.
- Use real numbers, real names, real examples in copy. If you don't know them, leave a `TODO:` comment naming the exact piece of content needed — don't invent filler.
- Add at least one **idiosyncratic detail per page**: a hand-drawn underline, a marginal note, a footnote, a rotated sticker, a grain/noise overlay, a marquee, a print-style caption. Things a template wouldn't ship with.
- Buttons: not all the same. A primary button can be a hard rectangle with an offset shadow. A secondary action can be a plain underlined link.

### 4. Motion
- Most things shouldn't animate. Page loads should be instant.
- When motion is used, make it specific: a number ticks up, an arrow nudges on hover, a chart draws once. Not blanket fade-ins.

### 5. Accessibility is non-negotiable
Whatever weird type or color choices you make, verify:
- Contrast ≥ 4.5:1 for body text, 3:1 for large text
- Focus rings are visible and on-brand (not the browser default, but not removed)
- Keyboard navigation works through every interactive element
- Reduced-motion preferences are respected

## Output format

For each change, give me:
1. **File path** and what you're changing
2. **The diff** (full updated component, not snippets)
3. **One sentence on why** — what AI-ish pattern this kills, what intent it replaces it with

Start with the audit. Don't rewrite anything until I've seen the audit and confirmed direction.
