====================================================
API DESIGN
====================================================

Create one main endpoint:

POST /analyze-frame

Input:
- image (base64 or multipart file)
- user_id
- exam_id
- timestamp (client time)

Output:
- camera_status: OK / WARNING / BLOCKED
- face_detected: true / false
- brightness_score
- frame_variance_score
- confidence_score (0–100)
- violation_type (if any)
- violation_severity (low / medium / high)

====================================================
CAMERA CHECK LOGIC (CRITICAL)
====================================================

For every snapshot:

1. BRIGHTNESS CHECK
- Convert to grayscale
- Calculate mean brightness
- If too dark or uniform → possible camera cover

2. FRAME FREEZE CHECK
- Compare with previous frame for same user
- If difference below threshold → frozen feed

3. FACE PRESENCE CHECK
- Use MediaPipe to detect face landmarks
- If face missing beyond allowed time → violation

4. OBSTRUCTION DETECTION
- Detect sudden drop in brightness
- Detect low edge density
- Detect uniform color patterns (tape / finger / cloth)

====================================================
CONFIDENCE SCORING
====================================================

Calculate confidence score using:
- Face detection confidence
- Brightness validity
- Frame variance
- Motion consistency

If confidence < threshold:
- Trigger violation

====================================================
VIOLATION RULES
====================================================

Trigger violation when:
- Camera covered
- Camera frozen
- Face missing
- Camera turned off

Violation handling:
- First → warning
- Repeated → severity escalation
- Persistent → BLOCKED status

====================================================
PRE-EXAM & DURING-EXAM BEHAVIOR
====================================================

- Before exam start:
  - If camera status ≠ OK → block exam

- During exam:
  - If camera status becomes BLOCKED:
    - Log violation
    - Notify frontend
    - Allow exam pause or escalation
