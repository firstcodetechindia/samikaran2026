Implement the MIRA AI Tutor credit system exactly as a metered usage model where students are allowed to freely consume credits until exhaustion, with no automatic fallback, no forced text-only mode, and no artificial restriction on AI behavior. Credits represent prepaid or plan-bundled AI usage value and are meant to be exhaustible by design. Both TEXT and VOICE interactions must operate under the same credit economy, fully abstracted from real API costs. TEXT responses should continue using the existing word-based slab system for user-facing deduction (1–100 words = 2 credits, 101–300 = 4 credits, 301–500 = 6 credits, 500+ = 8 credits). VOICE responses must deduct higher credits and internally follow a strict cost rule where every 1 character sent to Text-to-Speech consumes 2 internal credits (1 char = 2 credits internally), configurable only via admin controls and never exposed to users. Voice usage must be charged based on actual internal character usage plus existing text credits and premium logic, ensuring that higher usage always results in higher deduction and cost recovery. The system must NOT auto-disable voice, NOT shorten responses for cost reasons, and NOT intervene when usage is high; if a student continues using MIRA heavily, credits must naturally decrease and reach zero. When credits reach zero, the platform’s existing exhaustion and renewal behavior should trigger without introducing any new messaging, warnings, or UX changes. No credit, character, STT, TTS, ElevenLabs, or cost calculations should ever be shown to students. Admin must retain full control to adjust internal multipliers (such as 1 char = 2, 2.5, or 3 credits) for business protection, while students experience a simple, honest pay-for-usage system where more use equals more charge and exhaustion leads to optional renewal. This implementation must preserve current user experience while ensuring that AI cost is always covered by consumed credits and the system remains scalable and profitable at high usage volumes.
