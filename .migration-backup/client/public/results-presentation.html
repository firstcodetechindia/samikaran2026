1Results System Overview

Automated Results Calculation

The platform automatically calculates results when exam attempts are completed, ensuring accuracy and consistency across all participants.

  • Real-time score calculation as answers are submitted
  • Support for multiple question types with different scoring rules
  • Automatic negative marking for incorrect answers (configurable)
  • Time-based calculations for exam duration

Score Components

  • Total Score: Sum of marks obtained
  • Max Score: Maximum possible marks
  • Percentage: (Score / Max) x 100
  • Correct Answers: Count of right answers
  • Wrong Answers: Count of incorrect answers
  • Unattempted: Questions not answered

Negative Marking (Optional)

Not all olympiads use negative marking. When enabled:

  • Enable/disable per olympiad setting
  • Configurable deduction percentage
  • Applied only to wrong answers
  • Unattempted questions: No penalty

Score Calculation Formulas:

Without Negative Marking: Final Score = Correct x Marks Per Question

With Negative Marking: Final Score = (Correct x Marks) - (Wrong x Negative %)

Example without negative: 40 correct (2 marks each) = 80 marks
Example with negative: 40 correct - 10 wrong (25% of 2 = 0.5 each) = 80 - 5 = 75 marks

2Dual Ranking System

The platform supports two internationally recognized ranking mechanisms. Administrators can choose which ranking system to use when publishing results.

Standard Competition Ranking
1-2-2-4
Gaps appear after ties (Olympic style)
Dense Ranking
1-2-2-3
No gaps in ranking sequence

Example: 5 Students with Scores

Student Score Standard (1224) Dense (1223)
Ananya Singh 95/100 1 1
Rohan Sharma 90/100 2 2
Priya Patel 90/100 2 2
Arjun Kumar 85/100 4 3
Meera Reddy 80/100 5 4

Key Difference

In Standard Competition Ranking, if two students tie at Rank 2, the next rank is 4 (not 3). In Dense Ranking, the next rank is always consecutive (3), regardless of ties.

3Standard Competition Ranking (1-2-2-4)

How It Works

Standard Competition Ranking assigns the same rank to tied participants and then skips subsequent ranks equal to the number of tied positions.

Algorithm:

Next Rank = Current Position + Count of Ties at Current Rank

If 3 students tie at rank 2, the next rank is 5 (skipping ranks 3 and 4)

Detailed Example: 10 Students

2Student B952
3Student C952
4Student D952
5Student E905
6Student F856

8Student H808

10Student J759

When to Use

  • Olympic Games and sports competitions
  • Academic rankings where position matters
  • When total participant count should be preserved
  • Prize distribution based on exact standings

Advantages

  • Reflects true competitive position
  • Last rank equals total participants
  • Industry standard for competitions
  • Clear distinction between groups

4Dense Ranking (1-2-2-3)

How It Works

Dense Ranking assigns the same rank to tied participants but always uses consecutive integers, without gaps, for subsequent ranks.

Algorithm:

Next Rank = Current Rank + 1 (regardless of ties)

If 3 students tie at rank 2, the next rank is still 3

Detailed Example: Same 10 Students

Position Student Score Dense Rank
1Student A981
2Student B952
3Student C952
4Student D952
5Student E903
6Student F854
7Student G854
8Student H805
9Student I756
10Student J756

When to Use

  • Educational assessments and grading
  • When "levels" of achievement matter more
  • Scholarship tiers based on performance bands
  • Award categories (Gold, Silver, Bronze)

Advantages

  • Simpler to understand for students
  • No gaps in ranking numbers
  • Better for tier-based recognition
  • Maximum rank = number of unique scores

5Result Preparation Workflow

  1. Exam Completion: Wait for all participants to complete the exam or for the exam window to close.
  2. Auto-Calculation: System automatically calculates scores for all completed attempts.
  3. Review Period: Administrators can review calculated results and verify accuracy.
  4. Select Ranking Method: Choose between Standard Competition (1-2-2-4) or Dense (1-2-2-3) ranking.
  5. Apply Ranking: System applies selected ranking algorithm to all results.
  6. Quality Check: Verify top performers and edge cases (ties, zero scores).
  7. Approve Results: Authorize results for publication.
  8. Publish: Results become visible to students based on declaration date.

Important Notes

Once results are published, they cannot be modified without admin override. Always double-check rankings before final publication. The ranking method selection is irreversible after publication.

Tie-Breaking Rules (When Enabled)

  • Primary: Total score comparison
  • Secondary: Fewer wrong answers preferred
  • Tertiary: Time taken (faster completion wins)
  • Final: If still tied, same rank is assigned

6Technical Implementation

Database Schema

Results are stored in the olympiad_results table with the following key fields:

  • student_id: Reference to student registration
  • olympiad_id: Reference to the exam
  • total_score: Calculated score with negative marking
  • max_score: Maximum possible score
  • rank: Final assigned rank (Standard or Dense)
  • dense_rank: Dense rank for comparison
  • percentile: Calculated percentile position

Percentile Calculation:

Percentile = ((Total - Rank) / Total) x 100

Where Total = total number of participants, Rank = student's rank

API Endpoints

  • GET /api/olympiads/:id/results
  • POST /api/olympiads/:id/calculate-results
  • POST /api/olympiads/:id/approve-results
  • GET /api/students/:id/results

Security Measures

  • Role-based access control (RBAC)
  • Audit logging for all changes
  • Result tampering detection
  • Encrypted data transmission

Performance Optimization

For large-scale olympiads with thousands of participants:

  • Batch processing for result calculation
  • Indexed queries for rank computation
  • Cached result sets for repeated access
  • Asynchronous PDF certificate generation
