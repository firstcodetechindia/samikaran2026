1 Prerequisites - What You Need Before Starting
Before you start, make sure you have these things ready. This will save you time later.
Things You Need:
  • A computer with internet connection
  • A credit/debit card for AWS account (they won't charge unless you exceed free tier)
  • A domain name (like samikaran.com) - you can buy from GoDaddy, Namecheap, or Google Domains
  • Your project code (either on GitHub or in a zip file)
  • OpenAI API key for AI features
  • About 2-3 hours of time for complete setup
What is AWS?

AWS (Amazon Web Services) is like renting a computer in Amazon's data center. Instead of buying a physical server, you rent one online and pay only for what you use. We'll use two AWS services: EC2 (the server) and RDS (the database).

2 Create AWS Account & Setup IAM User
First, you need an AWS account. If you already have one, skip to the IAM user creation part.
A. Create AWS Account
Go to aws.amazon.com and click "Create an AWS Account"
  • Enter your email and choose a password
  • Enter your name and phone number
  • Add credit/debit card (for verification only)
  • Choose "Basic Support - Free" plan
  • Verify your phone number with OTP
B. Create IAM User (For Security)
IAM user is a separate login for daily use. Never use root account for regular work.
  • Login to AWS Console
  • Search for "IAM" in search bar and click it
  • Click "Users" in left menu, then "Create user"
  • Username: samikaran-admin
  • Check "Provide user access to AWS Management Console"
  • Select "I want to create an IAM user"
  • Set a password
  • Click Next, then "Attach policies directly"
  • Search and select: AdministratorAccess
  • Click "Create user"
  • Save the login URL, username, and password somewhere safe
From now on...

Always login using the IAM user account, not the root account. The root account should only be used for billing and emergencies.

3 Launch EC2 Server (Virtual Machine)
EC2 is your virtual server where the application will run. Think of it as a computer in the cloud that's always on.
A. Open EC2 Dashboard
  • Login to AWS Console with your IAM user
  • In the top right, click the region dropdown
  • Select "Asia Pacific (Mumbai) ap-south-1" - this is closest to India
  • Search for "EC2" in search bar and click it
  • Click the orange "Launch instance" button
B. Configure Instance
SettingValueWhy?
Namesamikaran-olympiadEasy to identify later
AMI (Operating System)Ubuntu Server 22.04 LTSFree, stable, well-supported
Architecture64-bit (x86)Standard choice
Instance typet3.medium2 CPU, 4GB RAM - good for 500+ users
C. Create Key Pair (Very Important!)
This is like a password file to connect to your server. You can only download it ONCE.
  • Click "Create new key pair"
  • Name: samikaran-key
  • Type: RSA
  • Format: .pem (for Mac/Linux) or .ppk (for Windows with PuTTY)
  • Click "Create key pair" - file will download automatically
  • SAVE THIS FILE SAFELY! Store it in a secure folder. You cannot download it again.
D. Network Settings (Firewall)
This controls who can access your server and on which ports.
  • Click "Edit" next to Network settings
  • Check "Allow SSH traffic from" - Select "Anywhere"
  • Check "Allow HTTPS traffic from the internet"
  • Check "Allow HTTP traffic from the internet"
  • Click "Add security group rule"
  • Type: Custom TCP, Port: 5000, Source: Anywhere (0.0.0.0/0)
E. Storage
  • Change storage from 8 GB to 30 GB
  • Keep type as gp3
F. Launch!
  • Review everything
  • Click "Launch instance"
  • Wait 2-3 minutes for instance to start
  • Click "View all instances"
  • Note down the Public IPv4 address (like 13.234.xxx.xxx)
Cost Alert

t3.medium costs approximately $30-40/month. You can start with t3.micro (free tier eligible) for testing, then upgrade later.

  • VPC: Default VPC
  • Public access: Yes (so EC2 can connect)
  • VPC security group: Create new
  • New security group name: samikaran-db-sg
  • Availability Zone: No preference
D. Additional Settings
  • Initial database name: samikaran
  • Backup retention: 7 days
  • Enable auto minor version upgrade
E. Create Database
  • Click "Create database"
  • Wait 10-15 minutes for database to be ready (Status: Available)
  • Click on database name
  • Copy the Endpoint (like samikaran-db.xxxxx.ap-south-1.rds.amazonaws.com)
F. Allow EC2 to Connect to Database
We need to allow our EC2 server to talk to the database.
  • In RDS dashboard, click on your database
  • Click on "VPC security groups" link
  • Click on the security group ID
  • Click "Inbound rules" tab, then "Edit inbound rules"
  • Click "Add rule"
  • Type: PostgreSQL, Source: Anywhere IPv4 (0.0.0.0/0)
  • Click "Save rules"
Save These Details

Write down: Database Endpoint, Username (dbadmin), Password, and Database name (samikaran). You'll need these later.

5 Connect to Your Server via SSH
SSH is a secure way to connect to your server from your computer. It's like remote desktop but for command line.
For Windows Users:
  • Download and install PuTTY from putty.org
  • If you downloaded .pem file, use PuTTYgen to convert to .ppk
  • Open PuTTY
  • Host Name: ubuntu@YOUR_EC2_IP
  • Go to Connection > SSH > Auth > Credentials
  • Browse and select your .ppk file
  • Click "Open"
  • Click "Accept" on security alert
For Mac/Linux Users:
Terminal
# First, set correct permissions on key file
chmod 400 ~/Downloads/samikaran-key.pem

# Connect to server
ssh -i ~/Downloads/samikaran-key.pem ubuntu@YOUR_EC2_PUBLIC_IP

# Type 'yes' when asked about fingerprint
You're Connected!

If you see "ubuntu@ip-xxx-xxx-xxx-xxx:~$" then you're successfully connected to your server. All following commands should be run in this terminal.

6 Install Required Software on Server
Your server is like a new computer - we need to install the programs our application needs to run.
A. Update System
First, update all existing software to latest versions.
Run on Server
sudo apt update && sudo apt upgrade -y
B. Install Node.js 20
Node.js is the runtime that executes our JavaScript code.
Run on Server
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# Verify installation
node --version   # Should show v20.x.x
npm --version    # Should show 10.x.x
C. Install PM2 (Process Manager)
PM2 keeps your app running even after server restart and handles crashes automatically.
Run on Server
sudo npm install -g pm2
D. Install Nginx (Web Server)
Nginx receives requests from internet and forwards them to your app. Also handles SSL.
Run on Server
sudo apt install -y nginx

# Start Nginx
sudo systemctl start nginx
sudo systemctl enable nginx
E. Install Python (for Proctoring Service)
Run on Server
sudo apt install -y python3 python3-pip python3-venv
F. Install Git
Run on Server
sudo apt install -y git
All Software Installed!

Your server now has everything needed to run the Samikaran Olympiad platform.

Now we need to get the application code onto the server. There are two ways to do this.
Option A: Clone from GitHub (Recommended)
If your code is on GitHub:

Then on Server
cd /home/ubuntu
unzip samikaran.zip
cd samikaran-olympiad
Install Dependencies
This downloads all the packages your application needs.
Run on Server
npm install
8 Configure Environment Variables
Environment variables store sensitive information like passwords and API keys. We create a .env file that the application reads.
Create .env File
Run on Server
nano .env
This opens a text editor. Paste the following (replace the highlighted values with your actual values):
.env content
DATABASE_URL=postgresql://dbadmin:YourSecurePassword123!@samikaran-db.xxxxx.ap-south-1.rds.amazonaws.com:5432/samikaran
NODE_ENV=production
PORT=5000
SESSION_SECRET=paste-a-64-character-random-string-here
OPENAI_API_KEY=sk-your-openai-api-key
ALLOWED_DOMAINS=samikaran.com,www.samikaran.com
Save and Exit nano
  • Press Ctrl + X to exit
  • Press Y to confirm save
  • Press Enter to confirm filename
Generate SESSION_SECRET
Run this command to generate a secure random string:
Run on Server
openssl rand -hex 32
Copy the output and paste it as SESSION_SECRET value in your .env file.
Environment Variables Explanation
VariableWhat It Does
DATABASE_URLConnection string to your RDS database
NODE_ENVTells app to run in production mode (faster, more secure)
PORTWhich port the app listens on (5000)
SESSION_SECRETSecret key for encrypting user sessions
OPENAI_API_KEYAPI key for AI features (question generation, chatbot)
ALLOWED_DOMAINSYour domain names (for CORS security)
9 Build and Start the Application
Now we compile the code and start the application. The build process converts TypeScript to JavaScript and bundles everything for production.
A. Build the Application
Run on Server
npm run build
This takes 2-5 minutes. Wait until you see "Build completed" or similar message.
B. Push Database Schema
This creates all the tables in your database.
Run on Server
npm run db:push
C. Start Application with PM2
Run on Server
# Start the application
pm2 start npm --name "samikaran" -- start

# Save PM2 configuration (so it restarts after server reboot)
pm2 save

# Setup auto-start on server boot
pm2 startup
# Copy the command it shows and run it
D. Verify Application is Running
Run on Server
# Check status
pm2 status

# View logs
pm2 logs samikaran

# Test if app responds
curl http://localhost:5000
App is Running!

If you see HTML output from the curl command, your application is running correctly on port 5000.

10 Setup Nginx (Web Server)
Nginx acts as a "front door" for your application. It receives requests on port 80/443 and forwards them to your app on port 5000.
A. Create Nginx Configuration
Run on Server
sudo nano /etc/nginx/sites-available/samikaran
Paste this configuration:
Nginx Config
server {
    listen 80;
    server_name samikaran.com www.samikaran.com;

    location / {
        proxy_pass http://localhost:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        proxy_read_timeout 300s;
        proxy_connect_timeout 75s;
    }
}
Save with Ctrl+X, then Y, then Enter.
B. Enable the Configuration
Run on Server
# Create symbolic link to enable site
sudo ln -s /etc/nginx/sites-available/samikaran /etc/nginx/sites-enabled/

# Remove default site
sudo rm /etc/nginx/sites-enabled/default

# Test configuration for errors
sudo nginx -t

# Restart Nginx
sudo systemctl restart nginx
Test It!

Open your browser and go to http://YOUR_EC2_IP. You should see your application! (without domain, just IP for now)

11 Point Your Domain to Server
Now we need to connect your domain name (like samikaran.com) to your server's IP address.
In Your Domain Registrar (GoDaddy, Namecheap, etc.)
  • Login to your domain registrar
  • Go to DNS settings / DNS Management
  • Find or create an A Record
  • Set Host/Name to @ (or leave blank)
  • Set Value/Points to YOUR_EC2_IP
  • TTL: 600 or Auto
  • Save
For www subdomain
  • Create another A Record
  • Host/Name: www
  • Value: YOUR_EC2_IP
  • Save
DNS Takes Time

DNS changes can take 5 minutes to 48 hours to take effect worldwide. Usually it's 15-30 minutes. You can check if it's working by running: ping samikaran.com

12 Install SSL Certificate (HTTPS)
SSL makes your site secure (https:// instead of http://). This is required for payments and shows a padlock in browser. We use Certbot which provides free SSL certificates.
Install Certbot and Get Certificate
Run on Server
# Install Certbot
sudo apt install certbot python3-certbot-nginx -y

# Get SSL certificate (replace with your domain)
sudo certbot --nginx -d samikaran.com -d www.samikaran.com
Certbot Questions
  • Enter email: Your email for certificate notifications
  • Agree to terms: Y
  • Share email with EFF: N (or Y if you want)
  • Redirect HTTP to HTTPS: 2 (Redirect - recommended)
Setup Auto-Renewal
SSL certificates expire after 90 days. Certbot auto-renews them.
Run on Server
# Test auto-renewal
sudo certbot renew --dry-run
HTTPS is Working!

Visit https://samikaranolympiad.com - you should see a padlock icon and your website should load securely.

13 Start Proctoring Service
The proctoring service is a Python application that monitors students during exams. It runs separately from the main application.
Setup and Start Proctoring
Run on Server
# Go to proctoring folder
cd /home/ubuntu/samikaran-olympiad/proctoring_service

# Install Python dependencies
pip3 install -r requirements.txt

# Start with PM2
pm2 start python3 --name "proctoring" -- main.py

# Save configuration
pm2 save
Verify Both Services Running
Run on Server
pm2 status
You should see two processes: "samikaran" and "proctoring" both with status "online".
14 Create Super Admin Account
Now you need to create the first admin account to manage the platform.
Create Super Admin
Run on Server or Browser
curl -X POST https://samikaran.com/api/admin/create-super-admin \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@samikaran.com","password":"YourSecurePassword123!"}'
You're Done!

Visit https://samikaranolympiad.com and login with your super admin credentials. You can now manage the entire platform!

15 Server Monitoring & Maintenance
Here are useful commands to monitor and maintain your server.
Daily Monitoring Commands
Common Commands
# Check if apps are running
pm2 status

# View application logs
pm2 logs samikaran
pm2 logs proctoring

# View logs in real-time
pm2 logs --lines 100

# Check server resources (CPU, Memory)
pm2 monit

# Restart application
pm2 restart samikaran
pm2 restart proctoring
pm2 restart all

# Check disk space
df -h

# Check memory usage
free -m
Updating the Application
When you have new code
cd /home/ubuntu/samikaran-olympiad

# Pull latest code (if using Git)
git pull

# Install any new dependencies
npm install

# Rebuild
npm run build

# Restart application
pm2 restart samikaran
Server Backup (RDS)
RDS automatically takes daily backups. You can also create manual snapshots:
  • Go to AWS RDS Console
  • Select your database
  • Actions > Take snapshot
  • Name it with date: samikaran-backup-2026-01-27
Monthly Tasks

1. Check AWS billing to ensure costs are as expected
2. Review RDS storage usage (upgrade if approaching limit)
3. Update Ubuntu: sudo apt update && sudo apt upgrade -y
4. Review PM2 logs for any recurring errors

Congratulations!

Your Samikaran Olympiad platform is now live on AWS! If you face any issues, check PM2 logs first (pm2 logs), then Nginx logs (sudo tail -f /var/log/nginx/error.log).
